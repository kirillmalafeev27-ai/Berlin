# Grünbach 2.0 — архитектура 2D-перезапуска

Игра-учебник немецкого A1: 2D-приключение, прохождение которого = полный курс
A1 по 16 модулям Übungsbuch (Wortschatz Module 1–16, ~970 слов и фраз,
`docs/wortschatz-modules-raw.txt`). Позиционирование: **«пройди игру — сдай
Goethe A1»**, для русскоязычных.

---

## 0. Конституция проекта (нерушимые правила)

Каждая фича и каждый диалог проверяются против этих восьми правил. Нарушение
любого из них — блокер релиза уровня.

1. **Прогрессивная замена языка**: реплики NPC начинаются по-русски и слово за
   словом становятся немецкими по мере освоения лексики игроком.
2. **SRS с 15+ типами вопросов**: фоновая интервальная система повторений.
3. **Экзамены-боссы** — гейты между главами: не сдал — дальше не идёшь.
4. **Тап по слову в реплике** → значение, произношение, грамматическая роль;
   для невыученных — «Понятия не имею, что это значит 🤷».
5. **Уровни владения словом** (mastery 0–5) управляют и заменой в диалогах,
   и частотой в SRS.
6. **Соперник** для внезапного повторения старого материала.
7. **NPC говорят только выученное** (+ явный «следующий шаг» сцены).
   ИИ-диалог проходит фильтр по освоенному словарю.
8. **Дизайн уровней вокруг переиспользования**: «введено → немедленно
   используется», каждый уровень максимально гоняет старый материал.
   **Запрет невыученных слов** обеспечивается линтером на этапе сборки.

## 1. Технологический стек

| Слой | Выбор | Почему |
|---|---|---|
| Рендер/мир | **Phaser 3** (WebGL/Canvas) | зрелый 2D-движок, тайлмапы, спрайты, веб |
| Карты | **Tiled** (.tmj JSON) | объекты с метаданными (vocabRef, quest, npc) прямо в редакторе |
| Язык | JS ES-модули (как сейчас) | переносим существующий код без переписывания |
| Сборка | Vite (уже в package.json) | dev-сервер, бандл |
| Сервер | **`server.mjs` без изменений** | /api/tts, /api/stt, /api/dialogue, статика — уже готовы |
| Голос | ElevenLabs TTS + alignment, Whisper/Scribe STT | **готовый конвейер из 3D-версии** |
| Деплой | Render (как сейчас) | render.yaml уже есть |
| Мобильные (этап 2) | Capacitor-обёртка | 2D-веб заводится на телефонах без переделки |
| Арт | пиксель-арт: тайлсеты 16/32px + **портреты NPC** (анимированные спрайт-листы) | язык жанра Pokémon/Stardew/Wagotabi |

Что переезжает из 3D-проекта как есть или почти как есть:
- `server.mjs` целиком (TTS с alignment, STT с двумя провайдерами и
  анти-галлюцинациями, ИИ-диалог с памятью/фактами/историей-репликами);
- логика квестовых автоматов (FSM гастхауса/рынка/пекарни) — шаблон для
  сценок глав;
- парсеры фраз + `normalizeText` + план «судьи фраз» (regex fast-path → LLM);
- эмодзи-словарь существительных (`GERMAN_NOUN_EMOJI`) — зародыш иллюстраций
  словаря;
- `stripPictographs`, скорость речи A1 (rate 0.74) и весь speak-конвейер;
- HUD текущего квеста, чипы-подсказки с шаблонами `___`;
- Dorfbuch → эволюционирует в Словарь+Журнал;
- e2e-методика тестирования через Playwright (боты-прохождения).

## 2. Мир: 16 модулей → 16 глав-локаций

Сеттинг — путешествие по Германии в Бремен (Deutsche Märchenstraße,
сказки Гримм как обрамление глав; публичное достояние, узнаваемо). Каждая
глава = город/район, тема города = тема модуля. Границы города = экзамен-босс.

| Гл. | Модуль (тема из PDF) | Локация | Босс-экзаменатор | Реюз готового |
|---|---|---|---|---|
| 1 | Willkommen! (приветствия, имя, откуда, buchstabieren) | деревня **Grünbach**, ворота | стражник **Bruno** — «пропуск в деревню» | квест стражника |
| 2 | Name, Adresse, Beruf | ратуша: Anmeldung (формуляр!) | чиновница — заполни формуляр со слуха | Dorfbuch-формуляр |
| 3 | Was ist das? (предметы, артикли) | школа, первая встреча с **сенсеем** | учительница Ida — der/die/das дуэль | Ida из квеста 3 |

| 5 | Tanzen oder wandern? (хобби) | парк/клуб | тренер — расскажи о хобби |  |
| 6 | Familie & Freunde | дом семьи попутчика | бабушка — «кто есть кто» по фото |  |
| 7 | Kaffee oder lieber Schokolade? (еда) | **Gasthaus** + кафе | Frau Berta — приём заказов | гастхаус-квест |
| 8 | Termine, Termine… (время, дни) | практика врача/офис | секретарь — назначь термин на слух |  |

Соперник (см. §7) путешествует тем же маршрутом и перехватывает игрока
между городами.

## 3. Модель данных (JSON, всё — статические файлы + localStorage)

### 3.1 `data/vocab/moduleNN.json` — словарь (из PDF автоматическим импортом)

```json
{
  "id": "m01_guten_morgen",
  "module": 1,
  "type": "chunk",              // word | chunk | grammar
  "de": "Guten Morgen.",
  "lemma": "Guten Morgen",
  "pos": "phrase",              // noun|verb|adj|adv|prep|phrase|pron|num
  "gender": null,               // "m"|"f"|"n" для существительных → der/die/das
  "plural": null,
  "ru": "Доброе утро.",
  "hint": "приветствие до ~11 утра",
  "illustration": "🌅",         // эмодзи на старте, спрайт позже
  "tags": ["greeting"],
  "audio": "audio/vocab/m01_guten_morgen.mp3",
  "examples": ["Guten Morgen, Frau Berta!"]
}
```

Пайплайн: `tools/import-wortschatz.mjs` парсит
`docs/wortschatz-modules-raw.txt` → 16 JSON-файлов; переводы/род/тип
дозаполняются LLM-скриптом **офлайн** (`tools/enrich-vocab.mjs`) и
проверяются вручную. Аудио: `tools/gen-audio.mjs` — батч в ElevenLabs
(existing key), кэш mp3 + alignment для липсинка портретов.

### 3.2 Прогресс игрока (`localStorage`, ключ `berlin2d.save.v1`)

```json
{
  "words": {
    "m01_guten_morgen": {
      "mastery": 3,                  // 0..5, см. §5
      "srs": { "ease": 2.4, "intervalDays": 3, "dueAt": 1760000000, "lapses": 1 },
      "stats": { "seen": 7, "correct": 5, "wrong": 2, "produced": 2 }
    }
  },
  "grammar": { "g_satzklammer": { "mastery": 2 } },
  "chapter": 4, "quests": {...}, "inventory": {...}, "money": 27,
  "achievements": ["first_word", "module1_boss"],
  "day": 12, "streakDays": 5
}
```

**Обязательное отличие от 3D-версии: прогресс персистентен.** Сохранение
после каждого события; экспорт/импорт файлом; позже cloud save.

### 3.3 Диалоги: `data/dialogues/chNN/*.json` — токенизированные реплики

Каждая реплика NPC — массив токенов со ссылками на словарь. Это ядро
прогрессивной замены и тап-глосс:

```json
{
  "id": "ch1_bruno_greet_1",
  "speaker": "bruno",
  "introduces": ["m01_guten_morgen"],       // «следующий шаг» этой сцены
  "tokens": [
    { "vocab": "m01_guten_morgen", "de": "Guten Morgen", "ru": "Доброе утро" },
    { "raw": ", " },
    { "vocab": "m01_der_herr", "de": "der Herr", "ru": "господин" },
    { "raw": "!" }
  ],
  "audio": "audio/lines/ch1_bruno_greet_1.mp3",
  "portraitState": "talking"
}
```

## 4. Прогрессивная замена языка (правило 1)

Рендер реплики, токен за токеном:

| mastery слова | как показывается токен |
|---|---|
| 0 (не введено) | **по-русски**, серым курсивом |
| 1 (введено) | по-немецки, с подчёркиванием-подсказкой (тап даёт всё) |
| 2–3 | по-немецки, лёгкая подсветка |
| 4–5 | по-немецки, как обычный текст |

Слова из `introduces` показываются по-немецки сразу (пара DE+RU в глоссе) —
это и есть «следующий шаг» comprehensible input. К концу модуля все реплики
главы полностью немецкие — язык мира отражает прогресс.

**Озвучка**: TTS всегда читает только немецкие токены реплики (русские
пропускаются); чем выше средний mastery реплики, тем ближе rate к 0.74
(нижний порог 0.6 для «свежих» реплик).

## 5. Уровни владения словом (правило 5)

| Ур. | Название | Как достигается | Эффект |
|---|---|---|---|
| 0 | не встречалось | — | RU в репликах, «Понятия не имею 🤷» при тапе |
| 1 | введено | сенсей/сцена ввела слово | DE в репликах + полный глосс |
| 2 | узнаю́ | 2 правильных recognition-ответа SRS | DE, тонкая подсветка |
| 3 | вспоминаю | 2 правильных recall (ввод/аудио) | обычный текст, SRS реже |
| 4 | произвожу | использовано в фразе-ответе (судья/STT) | допускается в вопросах боссов «без подготовки» |
| 5 | освоено | интервал SRS ≥ 21 день без ошибок | выпадает из активного SRS, остаётся у соперника |

Ошибки понижают SRS-интервал (не mastery ниже 1). Mastery — единственный
источник истины для замены (§4), фильтра ИИ (§8) и линтера (§9).

## 6. SRS-движок (правило 2)

Упрощённый SM-2: `interval = prev * ease`, ease 1.3–2.8, шаги 0д→1д→3д→7д→
14д→21д. Очередь `due`-слов доступна: (а) в любой момент через меню
(«review anytime»), (б) обязательной порцией утром игрового дня (5–10 слов,
подаёт соперник или сенсей), (в) вплетается в вопросы боссов (30% старого).

**Типы вопросов (17):**
1. DE → RU, выбор из 4
2. RU → DE, выбор из 4
3. Аудио → выбор слова
4. Аудио → ввод с клавиатуры (диктант)
5. Картинка/эмодзи → выбор DE
6. DE → выбор картинки
7. Ввод перевода DE с клавиатуры
8. Собери предложение из плиток (порядок слов, Satzklammer)
9. Cloze: вставь пропуск (артикль/окончание/предлог)
10. der/die/das — выбор рода
11. Спряжение: ich/du/er-форма глагола
12. Найди лишнее (odd one out по тегу)
13. Сопоставь пары DE↔RU/картинка (matching)
14. **Произнеси фразу — STT-проверка** (уникально наше)
15. Ответь NPC полной фразой — судья фраз (regex → LLM)
16. Числа/цены на слух → ввод цифрами
17. Buchstabieren: собери слово из букв на слух

Каждому типу — вес по mastery: recognition-типы (1–6) качают 1→2,
recall (7–13, 16–17) качают 2→3, production (14–15) качают 3→4.

## 7. Боссы-экзамены и соперник (правила 3, 6)

**Босс** = персонаж главы (таблица §2). Формат: 15–25 вопросов, микс
70% текущий модуль / 30% предыдущие (interleaving), включая минимум
2 production-вопроса (STT или фраза-судья). Порог 80%. Провал: мягкий —
босс отправляет к сенсею на разбор ошибок (персональная ревью-сессия из
промахов), ретрай без штрафа. Успех: печать в паспорт путешественника
(= ачивка), открытие дороги в следующий город, катсцена.

**Соперник** («Max», едет тем же маршрутом в Бремен): появляется по
триггерам — вход в новую локацию, выход из экзамена, случайно на дорогах
(cooldown 1 игровой день). Дуэль: 3–5 вопросов **только из прошлых модулей**
(mastery ≥ 2), быстрый темп. Победа — монеты/предмет; поражение — ничего
страшного, но Max дразнится (мотивация). Это ходячий spaced retrieval.

## 8. NPC говорят только выученное + ИИ-фильтр (правило 7)

- **Скриптовые реплики**: гарантируется линтером (§9) на этапе сборки.
- **ИИ-свободный разговор** (зоны «поболтать», второй разговор с NPC):
  1. в system prompt передаётся список разрешённых лемм (модули ≤ N) +
     инструкция «используй ТОЛЬКО эти слова»;
  2. **пост-фильтр**: ответ лемматизируется (упрощённо: словарь форм из
     vocab + LLM-лемматизация оффлайн-кэшем); токены вне освоенного →
     реплика перегенерируется (1 ретрай) или заменяется скриптовым
     фолбэком «Das lernst du später! 😉»;
  3. память диалога и факты игрока — **уже реализовано** в server.mjs,
     переносится как есть.

## 9. Линтер контента (правило 8, «запрет невыученных слов»)

`tools/lint-dialogues.mjs` — запускается в CI и перед деплоем:

- каждый диалог объявляет `chapter`; допустимы токены: `vocab` из модулей
  ≤ chapter, `introduces` текущей сцены, имена собственные из белого списка;
- ошибка сборки, если найден DE-токен вне допуска или токен без `vocab`-ссылки;
- отчёт переиспользования: для каждой главы — % токенов из старых модулей
  (**норма ≥ 50%**), список слов модуля, НЕ использованных в главе
  (норма: 0 — «введено → немедленно используется»), покрытие каждым словом
  минимум 3 сцен главы введения и ≥ 1 сцены в двух следующих главах.

Это машинная гарантия методики, а не пожелание.

## 10. Тап по слову и словарь (правило 4)

**Тап по токену реплики** → карточка-поповер: DE (крупно, с артиклем и
цветом рода: der-синий/die-красный/das-зелёный), произношение (кнопка 🔊,
mp3 из кэша), RU-значение, грамматическая роль в ЭТОМ предложении
(«Akkusativ, прямое дополнение», «глагол, 2-е место»), уровень владения
(прогресс-точки), кнопка «в повторение». Для mastery 0: **«Понятия не имею,
что это значит 🤷»** и ничего больше (мотивация выучить).

**Словарь (Wörterbuch)**: списки по модулям и тегам (еда, транспорт…),
поиск, иллюстрация (эмодзи → спрайты позже), произношение, примеры,
статистика слова. Это эволюция Dorfbuch. **Kanjidex-аналог**: коллекция
«грамматических печатей» — каждая изученная конструкция (Satzklammer,
Akkusativ, du/Sie…) как красивая карточка-трофей.

## 11. Сенсей (грамматические уроки)

Персонаж-наставник — **сова Frau Eule** (или учительница Ida) — появляется
в начале каждой темы: карточка-урок = правило по-русски (3–4 строки, как
`RUSSIAN_LESSON_EXPLANATIONS` сейчас) + 2–3 озвученных немецких примера +
**немедленное применение**: 1 вопрос типа 8/9 сразу + первый квест главы
использует конструкцию. Уроки собраны в справочник «Grammatik» (re-read
anytime).

## 12. Диалоговый экран с портретами (presence)

Требование: разговор как «присутствие», а не текст в углу.

- **Портрет NPC крупным планом** (⅓ экрана, стиль visual novel поверх
  2D-мира): анимированный спрайт-лист (GIF-подобный цикл), состояния:
  `idle`, `talking` (рот двигается), `happy`, `thinking`. Смена кадров
  `talking` синхронизируется с **alignment из ElevenLabs** (конвейер уже
  есть в lipsync.js — переиспользуем тайминги, вместо морфов меняем кадры).
- Реплика — токенизированный текст с прогрессивной заменой и тап-глоссами.
- **Кнопки под репликой: «🔁 Повторить» и «🐢 Медленнее»** — повторное
  проигрывание mp3; «медленнее» = rate 0.6 (speak-конвейер уже поддерживает
  rate). Плюс кнопка 🎤 (STT-ответ) и поле ввода — как сейчас.
- Ответы игрока: чипы с пропусками `___` (не готовые фразы!), свободный
  ввод, голос.

Портреты: 8–12 персонажей × 4 состояния. Производство: спрайт-листы
(пиксель-арт 128×128, 4–6 кадров на состояние).

## 13. Мир, взаимодействия, экономика

- **Тап по объекту мира** (Tiled-объект с `vocabRef`): слово + артикль +
  звук; первый тап даёт mastery 0→«встречено» (в словарь, серым);
  соответствие правилу «осмотр предметов даёт слова». На каждой карте
  10–20 таких объектов из лексики главы.
- **Квесты-языковые задачи**: прогресс ТОЛЬКО через язык (заказать шницель,
  спросить дорогу, записаться на приём). Никаких «кликни по предмету» как
  условие прогресса (урок из пекарни).
- **Инвентарь**: рюкзак (билеты, покупки, подарки, письма) — предметы
  завязаны на квесты; **деньги (евро)** — заработок с дуэлей/квестов,
  траты в магазинах = тренировка чисел и цен (модуль 4).
- **Культурные мини-игры**: по одной на главу-праздник (Weihnachtsmarkt,
  Karneval) — механика простая, весь текст из освоенной лексики.
- **Ачивки и секреты**: печати городов, «все слова модуля на 5»,
  «победи Макса 5 раз», скрытые NPC-сказочные персонажи (Гриммы) с
  секретными словами, 100%-словарь главы. Экран достижений.

## 14. Прогресс-трекинг

Экран «Мой путь»: карта Германии с маршрутом и печатями; график слов по
уровням владения (стопки 0–5); статистика по модулям (слова/грамматика/
покрытие); streak дней; кнопка «Повторить сейчас» (review anytime);
прогноз: «до экзамена Goethe A1 осталось N слов и M конструкций».

## 16. Дорожная карта

| Этап | Содержимое | Критерий готовности |
|---|---|---|
| 0. Данные | import-wortschatz + enrich: 16 vocab-JSON; линтер | все ~970 слов в JSON, линтер бьёт тестовый диалог с нарушением |
| 1. Ядро | Phaser-мир (1 карта), диалоговый экран с портретом, «повторить/медленнее», прогрессивная замена, тап-глосс, персистенс | глава 1 (Willkommen!): 64 слова вводятся и используются, реплики русеют→немечатся |
| 2. Обучение | SRS + 10 типов вопросов, сенсей, словарь-UI, прогресс-экран | цикл «урок → квест → ревью» работает на главе 1 |
| 3. Гейты | босс Bruno (экзамен модуля 1), соперник Max, ачивки | главу 1 нельзя покинуть без 80% |
| 4. Масштаб | главы 2–4 (реюз ратуша/школа/рынок), остальные типы вопросов, STT-вопросы, судья фраз | 4 главы проходимы e2e-ботом |
| 5. Полировка | мини-игры, инвентарь/деньги, ИИ-фритолк с фильтром, мобильная обёртка | публичная бета |
| 6. Контент | главы 5–16, финальный Goethe-экзамен в Бремене | полный A1 |

Правило разработки, вынесенное из 3D-опыта: **каждый этап закрывается
автоматическим e2e-прохождением** (Playwright-бот проходит контент и
проверяет состояние), как мы делали с квестами 3D-версии.
